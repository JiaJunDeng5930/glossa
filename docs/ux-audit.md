## 产品原则

Glossa 通过紧凑、直接、可预测的交互引导用户。页面内只显示当前动作需要的最少信息，详细语义进入原生提示、辅助技术标签、设置项名称和工程注释。

本轮确认的核心语义：

- 释义成功展示并被用户继续阅读，代表用户认识该词，词汇进入 `known`。
- 制卡代表用户开始学习该词，词汇在学习窗口内保持可见，到期进入 `known`。
- 页面缓存保证同一页面反复开关翻译时结果稳定。
- 按住制卡快捷键进入选词状态；期间出现第二个按键时退出长按状态，组合键继续按其自身语义处理。
- AI 连接测试确认配置的传输端点可以接收请求；真实释义和制卡请求各自校验业务响应。
- 页面内 `...` 表示正在制卡，`✓` 表示完成，`×` 表示失败。
- 设置页采用显式保存，让一组相关调整作为一次变更提交。

## 回归覆盖

本轮回归测试覆盖以下用户路径：

- 多义词在不同句子中的卡片内容缓存与单词级重复确认。
- 跨行内标签的完整句子解析、页面原始文本保持与手动制卡上下文。
- 高阶词表的累计包含关系。
- 设置变更、缓存清理与页面内存失效。
- 弹窗的开启、停止、受限页面和中文错误状态。
- 设置页 provider 能力、词汇管理、卡片历史重置，以及 305px / 320px 窄屏布局。
- 首次设置的 AI 验证门槛、Anki 跳过、返回、重开、安装入口和保存失败反馈。
- 页面徽标与重复确认的可访问状态。

发布验收使用 `npm run verify`、构建后浏览器视觉检查和 `codex-review-final main`。
